Sphere 仕様書（ドラフト v0.1）
0. 目的

本システムは、球体入れ子＝情報圏/プラットフォーム圏として定義し、都市（地理）上に住む人々の 信頼・所属・情報摂取 が、分断・治安・制度・国家間緊張などのマクロ状態へ伝播する世界をシミュレートする。

1. コア概念
1.1 地理（City / World Surface）

**都市は“地理”**であり、人・企業・資源・治安などが存在する居住空間。

都市は 一意な地理ID（cityUid） を持ち、世界全体で一つの実体として扱う。

1.2 球（Sphere = 情報圏 / プラットフォーム圏）

**球は“情報が流れる世界（見えている現実）”**であり、都市を複製しない。

球はプラットフォームの挙動（推薦・検閲・拡散摩擦・ボット圧など）をパラメータとして持つ。

入れ子は「外側ほど広域で、内側ほど濃密（刺さる）」という性質を表す。

1.3 レイヤー（Layer0/1/2）

レイヤーは居住空間ではなく 役割（role） の分類。

Layer0：個人（Person） … 都市に住み、情報圏に接続し、意思決定する

Layer1：コミュニティ（Community） … 都市を跨ぐ所属・規範・門番・語り部などを持つ

Layer2：制度/プラットフォーム/国家（Institution/Platform/Gov） … 球のルール・規制・執行を司る

同一人物が複数レイヤーの帽子をかぶってよい（例：会社員兼コミュニティ門番）。

2. エンティティ仕様
2.1 City（地理都市）

主キー

cityUid: string

状態（例）

population

economyIndex（雇用/物価などの合成でも可）

securityIndex（治安）

polarizationIndex（分断度）

socialTrustIndex（一般信頼）

migrationIn/out（移住フロー）

2.2 Sphere（情報圏）

主キー

sphereId: string

nestLevel: int（外=0, 内ほど大）

プラットフォーム設定（最小セット）

rankingPolicy：neutral | outrage_boost | health_boost

moderationStrength：0..1

shareFriction：0..1（共有のしやすさの逆）

botPressure：0..1

credibilityWeight：0..1（権威重視度）

crossSphereFriction：球間移動/情報越境コスト

都市×球の投影状態（ビュー）

CitySphereState[cityUid][sphereId] を別テーブル/Mapで持つ（都市は複製しない）

例：rumorRate, trustDecay, feedMix, localNarrativeBias

2.3 Person（個人 / Layer0）

主キー

personId: string

属性

homeCityUid

sphereAffinities: Map<sphereId, float>（情報摂取比率）

beliefVector（少数次元でよい：秩序志向/反体制/陰謀耐性など）

attentionBudget（1日の摂取上限）

influence（発信力）

trustGraph（対象タイプ別：人物/コミュニティ/媒体）

行動（代表）

consume(info)

share(info)

verify(info)

report(info)

persuade(target)

join/leave(community)

migrate(city)

2.4 Community（コミュニティ / Layer1）

主キー

communityId: string

属性

type：company | religion | neighborhood | hobby | ...

cohesion（結束）

gatekeeping（門番強度）

normStrength（規範/制裁）

outgroupPenalty（外集団への不信・敵視）

roles（長老/司祭/HR/語り部/門番…）

ネットワーク

都市を跨げる（memberCityUids またはメンバーの居住都市で暗黙表現）

2.5 Institution / Platform / Gov（Layer2）

主キー

institutionId: string

属性

jurisdiction（都市集合、国境など）

regulationStrength（規制強度）

enforcement（執行）

officialNarrativeBoost（公式物語の増幅）

crisisMode（非常事態フラグ）

3. 情報（Information）モデル
3.1 Info（情報単位）

infoId

truthValue：0..1（真偽）

emotion：怒り/恐怖/快感 など（少数次元）

topic：宗教/治安/外交/経済…

sourceType：official | influencer | community | anonymous | bot

sphereOrigin（発生球）

cityOrigin（発生都市）

3.2 誤情報の3類型（必須）

誤情報（単純な間違い）

扇動（情動増幅で行動誘発）

偽の権威（専門家っぽさ）

4. コミュニケーション仕様
4.1 会話/伝播の種別

同レイヤー：L0↔L0、L1↔L1、L2↔L2

異レイヤー：L0↔L1、L1↔L2、L0↔L2

放送（1→N）：声明・インフル発信・ボット拡散

観測（N→1）：通報・監視・捜査

4.2 可達性ルール（球ごとに決める）

edgeAllowed[sphereId][fromType][toType] = {prob, cost, latency}

例：外側球は L0↔L0 高、L0→L2 低（BANはあるが対話は薄い）

内側球は L0↔L1 濃い、異端へのコスト高

6. 国家間争い・宗教利用（イベント仕様）
6.1 起点の型（シナリオテンプレ）

L2起点（国家主導）：制裁/検閲/規制切替 → 生活変化

L1起点（代理戦）：宗教/企業/共同体が衝突固定 → 国家が後追い

L0起点（炎上）：偽情報/偽映像 → 群衆暴走 → 制度が追随

ハイブリッド（推奨）：L2が設計→L0が引き金→L1が固定

6.2 宗教が“使われる”メカ（最小3つ）

正当化：暴力心理コスト低下

境界硬化：内集団信頼↑ 外集団信頼↓

異端狩り：内部多様性↓ 極端化↑

6.3 情報圏の分断イベント（球操作）

sphereSplit(A,B)：相互接続コスト↑、情報越境↓

trustShock(group)：特定集団への信頼急落

mobilization：結束↑、外集団ペナルティ↑
