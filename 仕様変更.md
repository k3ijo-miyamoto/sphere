# 仕様変更: ショックの歪み保存と相転移導入

## 0) 目的（世界観）
- 目的は「ショック増幅」ではなく、ショックを`latent strain`として保存し、臨界時に制度・移住・都市・地政が同時に変質する世界を作ること。
- 平常時は従来の安定性（TestA/Cで得た挙動）を維持する。
- ショック後は指標が戻っても歪みが残り、条件を満たすとレジーム切替が起きる。

## 1) 追加データ構造
- `city.strain: number (0..1)`
  - 都市に蓄積する潜在歪み。
- `city.regime: "normal" | "stressed" | "fractured"`
  - 都市モード。
- `world.systemState.strainMetrics`
  - `mean, p95, numStressed, numFractured, regimeTransitionsCount, residualStrainCases, transitionEvents`。
- `world.systemState.systemicTriggers`
  - `forceGeopoliticsTick, reason, transitionsToday`。
- `world.systemState.migrationRegimeStats`
  - regime別`attemptsByRegime/movesByRegime`。

## 2) 歪み蓄積ルール
- 毎tickで次式を適用:
  - `strain = clamp01(strain*(1-decay) + shockIntensity*k_add - recovery*k_relief + displacement)`
- `decay`は小さく（デフォルト `0.004/tick`）。
- `shockIntensity`は加重和:
  - 失業、物価（生活費）、不安定性、格差、信頼低下、資源ストレス、地政緊張、所得ストレス。
- `recovery`は安全・信頼・生産性・雇用・資源の回復力。
- 政策介入の副作用（`policyStrainDisplacement`）を追加し、短期指標改善でも歪みが完全には消えない。

## 3) 臨界＋ヒステリシス
- 遷移条件（デフォルト）:
  - `normal -> stressed` : `strain > 0.42`
  - `stressed -> fractured` : `strain > 0.72`
- 戻り条件:
  - `stressed -> normal` : `strain < 0.32`
  - `fractured -> stressed` : `strain < 0.58`
- レジーム別効果:
  - `trustRecoveryMult`
  - `instabilityAmplifier`
  - `migrationMult`
  - `hiringRecoveryMult`

## 4) ショック経路（最小4配線）
- 生活直撃: `economy/price/unemployment -> strain -> migration`
- 都市財政・雇用: `employment/profit/capacity -> strain/regime -> hiring recovery slowdown`
- 正統性: `inequality + trustDown -> instabilityUp -> strainUp`
- 外部ショック: `geopolitics tension/war -> strain`（地政由来ショックをstrain入力へ合流）

## 5) 政策/RLの扱い
- 政策は短期指標を改善可能。
- ただし副作用として`displacement`を通して歪みを残しうる（短期勝利・長期歪み）。
- 都市政策は既存の`Policy Genome + 軽量Bandit`を維持し、歪み保存設計と両立。

## 6) 更新頻度（多層同時更新）
- 都市でレジーム遷移が発生、または`fractured`比率が閾値超過した場合:
  - `world.systemState.systemicTriggers.forceGeopoliticsTick = true`
- `updateGeopolitics`に`forceUpdate`を追加し、夜間以外でも臨界時に更新可能。

## 7) 計測
- 追加メトリクス:
  - `strain_mean`, `strain_p95`, `num_stressed`, `num_fractured`
  - `regime_transitions_count`
  - `residualStrainCases`（指標回復後も歪みが残るケース）
  - `migrationRegimeStats`（regime別移住発火）

## 8) 受け入れ条件
- 平常時: 従来安定性を維持。
- ショック後: 経済指標が戻っても`strain`残存ケースが観測される。
- 臨界時: `regime transition`に同期して移住・都市不安定・地政更新が増える。
- 常時破滅にはならず、`fractured`は限定的で回復は遅い。

## 9) 実装タスクと対応
1. `city.strain/city.regime`追加
2. tickごとのstrain更新関数追加
3. ヒステリシス遷移追加
4. regime係数をmigration/trust/instability/hiringに反映
5. 臨界時の地政強制更新トリガー追加
6. 観測メトリクス追加
7. A/Cを壊さない前提で長期ラン検証

## 追記: TestB改善のための追加調整（臨界移住・局所地政・戻り配線）

### 調整1: 移住を「差」だけでなく「臨界」駆動に変更

#### 変更方針
- 既存の `base_migration = f(attractiveness_gap)` は維持。
- 追加で `strain/regime` 直結の `panic_migration` を導入。
- 全都市が悪い局面でも、差が小さくても逃避移住が発火するようにする。

#### 実装
- 移住発火確率:
  - `moveProb = clamp(baseMigration + panicMigration, 0, 0.92)`
  - `panicMigration = 0.18 * sigmoid((homeStrain - 0.48)/0.12) * regimeMult`
  - `regimeMult`: normal=1.0, stressed=1.45, fractured=2.0
- panicが一定以上のとき、`bestId==home`でも「相対的にマシ」な候補へ確率選択（softmax）を実行。
- 国跨ぎ移住フローを `world.systemState.migrationFlows.pairDaily/pairEma` に記録し、地政入力へ接続。

#### 反映ファイル
- `src/sim/population.js` (`applyRelocationMigration`)

### 調整2: 地政を「局所イベント＋結合」で上げる

#### 変更方針
- 平均テンション依存だけでなく、局所ホットスポットを直接driftへ注入。
- fractured都市・国際移住フロー・エネルギー価格ショックを入力に採用。
- 隣接（外交隣接）へのspillを追加。
- 臨界イベント時（force update）にburstを加算。

#### 実装
- Nation hotspot:
  - fractured比率、stressed比率、平均strain、cross-nation migration flow、energy shock を合成。
- Relation driftへ追加:
  - `localShock + spill + pairFlow + energyShock + burst`
- `forceUpdate`時に burst（+0.07）を注入。

#### 反映ファイル
- `src/sim/geopolitics.js` (`computeGeopoliticalHotspotInputs`, relation drift更新)

### 調整3: 地政→都市の戻り配線（最小1本: 価格）

#### 変更方針
- 地政テンションを都市へ価格/生産性/信頼の形で戻す。
- その価格ショックをstrain入力へ再接続して閉ループ化。

#### 実装
- 都市側へ `city.lifecycle.geoPriceShock` を更新。
- 都市メトリクスへ反映:
  - `costOfLiving += geoPriceShock * 0.006`
  - `productivity -= geoPriceShock * 0.004`
  - `trust -= geoPriceShock * 0.002`
- strainのshockIntensityに `geoPriceShock * 0.1` を追加。
- 国際エッジに対し、pair shockが高い場合は `permit/sealed` を適用し、border restrictionを増加させる。

#### 反映ファイル
- `src/sim/geopolitics.js` (`applyGeopoliticalFeedbackToCities`)
- `src/sim/cityDynamics.js` (`updateCityStrainAndRegime`)

### 想定効果
- `cityDestabilize -> migrationShift` が同一エネルギー源（strain）で接続。
- 臨界時にのみ局所地政ショックが立ち上がり、`geopoliticalSpill` を点火。
- 地政→都市→strainの戻りで `feedbackToCities` を発火しやすくする。

## 追記: 適用パッチ原文（2026-03-02）

### パッチ①：feedbackToCities を“差分が残る形”で確実化（Issue2最優先）

- 目的: `borderRestrictedEdges > 0` を発火後に残りやすくする。
- 実施:
  - `applyBorderPolicyByDiplomacy` と `applyGeopoliticalFeedbackToCities` を最終合成。
  - 最終edge状態を `max(外交由来, ショック由来, 既存状態)` で決定。
  - 優先順位は `sealed > permit > open`。
  - ショック由来制限にヒステリシス導入。
    - 発火: `pairShock > 0.16 / 0.32`
    - 解除: `pairShock < 0.10 / 0.20`

### パッチ②：economyShock が消える“過剰回復”を止める（Issue1/3）

- 目的: ショック局面で回復が勝ちすぎる現象を抑える。
- 実施:
  - `stressed/fractured` で `relief` を減衰。
  - ショック窓（20日相当tick）を導入し、窓中は `relief` をさらに減衰。
  - `decay` もレジームに応じて減速し、歪み保存を強化。

### パッチ③：TestCの頭打ち対策（Issue4）

- 目的: `policyRevisionCount` の飽和で差分が消える問題を解消。
- 実施:

  - 評価スクリプトに `policyRevisionRate` を出力。
  - TestCの `institutionPushback` 判定を `policyRevisionRate` 差分中心に変更。

### ついで対応：地政指標飽和（Issue5）

- `geopoliticalSpill` 判定を平均テンション依存からイベント寄りへ変更。

## 追記: 境界制限TTLロック + 最終確定統合 + 評価指標改修（2026-03-02）

## 追記: Ownership-based Wealth Dynamics 導入（2026-03-02）

### 目的

- 企業オーナーシップを個人資産に直接反映し、企業家・投資家の上方移動を表現しやすくする。
- 富の集中を観測可能にし、分配構造を継続評価できるようにする。

### 追加データ構造

- `company.valuation`
  - 企業価値（非線形更新）。
- `company.equityHolders`
  - 持分保有者一覧（`holderKey`, `shares`, `ownershipPct`）。
- 個人側（`person.socioeconomic`）に以下を追加:
  - `salaryWealth`
  - `equityWealth`
  - `wealthRaw`

### 富の算出式（更新）

- 仕様式:
  - `person.wealthRaw = salaryWealth + equityWealth`
  - `person.wealth = clamp01(person.wealthRaw)`
- 実装対応:
  - `salaryWealth` は所得からEMA更新。
  - `equityWealth` は `Σ(ownership × company.valuation)` を正規化して更新。

### 企業価値の更新（非線形）

- `company.growthExpectation` を profit と profitDelta から更新。
- `company.valuation` は以下の構造で更新:
  - `base = capital * w_cap + max(0, revenue) * w_rev`
  - `boost = (1 + posProfit * k_profit)^a * (1 + growthExpectation^k_growth)`
  - `drag = 1 + loss * k_loss`
  - `valuation = clamp(base * boost / drag, min, max*4)`

### Hyper Growth Event（希少・高倍率）

- 低確率で `hyperGrowthEvent` を発火。
- 発火時:
  - `hyperGrowthBoost` に大倍率を付与。
  - `valuation` に乗算。
- その後:
  - `hyperGrowthBoost` は tick ごとに減衰。

### 観測ログ追加

- `economy.wealthDistribution` を追加:
  - `gini`
  - `top1SharePct`
  - `top10SharePct`
  - `top20SharePct`
  - `min`, `max`, `avg`

### MCP表示拡張

- `sphere_company_financials` に以下を追加:
  - `valuation`
  - `equityHolders`（先頭上位）
- `sortBy` に `valuation` を追加。
