# 仕様変更3: 企業タイプ拡張（IT / Military）と安定化設計

## 1. 目的
- 既存企業モデルに `IT` と `Military` を追加し、経済・雇用・地政学の相互作用を強化する。
- ただし、富の暴走集中や戦争インセンティブの暴走を防ぐため、制度的な抑制を同時導入する。
- 段階導入で既存世界を壊さず、A/B比較可能な形で拡張する。

## 2. 導入方針（最小変更）
- 既存 `company` 構造は維持し、差分は `companyType` と係数群で表現する。
- 既存セクター（Industry/Finance/Logistics/Tech/etc）を置換せず、上位分類として追加する。
- まずは「収益関数」「採用選好」「政策効果」「地政学リンク」の4点を実装する。

## 3. データモデル変更

### 3.1 Company
- 追加:
  - `companyType: "General" | "IT" | "Military"`
  - `rdStock: number`（0..1）
  - `networkEffect: number`（0..2）
  - `compliance: number`（0..1）
  - `lobbyPower: number`（0..1）
  - `defenseContractShare: number`（0..1, Military中心）
  - `concentrationPenalty: number`（0..1）
- 既存互換:
  - 未設定時は `companyType="General"` として扱う。

### 3.2 Nation / Geopolitics state
- 追加:
  - `defenseBudget: number`（0..1）
  - `itRegulationLevel: number`（0..1）
  - `militaryExportControl: number`（0..1）

### 3.3 Config（defaultConfig）
- 追加セクション `companyTypes`:
  - `enabled: boolean`
  - `it`: 係数群（R&D効率、ネットワーク効果上限、規制感応度）
  - `military`: 係数群（緊張感応度、戦時倍率、制裁感応度）
  - `safety`: 上限係数（profitCap, valuationCap, concentrationCap）
- 追加セクション `policy`:
  - `antitrustStrength`
  - `defenseProcurementCap`
  - `dividendTaxRate`

## 4. 企業生成仕様

### 4.1 初期生成
- 既存 `createCompany()` に `companyType` 割当を追加。
- 都市タイプ別初期分布（初期案）:
  - `workHub`: General 70%, IT 20%, Military 10%
  - `mixed`: General 80%, IT 18%, Military 2%
  - `residential`: General 95%, IT 5%, Military 0%

### 4.2 日次創業補充
- 既存の最低社数補充ロジックを維持。
- 補充時の `companyType` は都市分布と地政学状態で重み調整:
  - 緊張高 + 防衛予算高 -> Military重み上昇
  - 教育・技能高 + IT規制低 -> IT重み上昇

## 5. 収益関数（中核）

### 5.1 IT企業
- 収益:
  - `ITRevenue = baseRevenue * talentFactor * rdFactor * networkFactor * regulationPenalty * concentrationPenalty`
- 構成:
  - `talentFactor`: 従業員の `cognitive/education` 加重平均
  - `rdFactor`: `1 + rdStock * k_rd`
  - `networkFactor`: `1 + min(networkEffect, cap_network)`
  - `regulationPenalty`: `1 - itRegulationLevel * k_reg`
  - `concentrationPenalty`: 反独占ペナルティ

### 5.2 Military企業
- 収益:
  - `MilitaryRevenue = baseRevenue * defenseBudgetFactor * tensionFactor * warMultiplier * sanctionPenalty * procurementCapPenalty`
- 構成:
  - `defenseBudgetFactor`: 国家 `defenseBudget` に比例
  - `tensionFactor`: 外交緊張平均に比例
  - `warMultiplier`: `status=war` の関係数に応じ上昇
  - `sanctionPenalty`: 制裁・輸出規制に応じ低下
  - `procurementCapPenalty`: 公共調達上限超過時の減衰

### 5.3 安全上限（必須）
- 利益・評価額のソフトキャップ:
  - `profit = clamp(profit, -capLoss, capProfit)`
  - `valuationGrowth = min(valuationGrowth, capValuationGrowth)`
- 企業集中ペナルティ:
  - 都市/セクター別 HHI が閾値超過で収益減衰。

## 6. 雇用・賃金仕様

### 6.1 採用選好
- IT:
  - `hireScore += cognitive*0.4 + education*0.35 + discipline*0.15 + openness*0.10`
- Military:
  - `hireScore += discipline*0.35 + stressResilience*0.3 + health*0.2 + conformity*0.15`

### 6.2 賃金プレミアム
- ITプレミアムとMilitaryプレミアムを段階制で付与。
- 上限を持つ（都市平均賃金比で最大倍率を設定）。

## 7. 政策・制度連動

### 7.1 IT政策
- `antitrust`: HHI高企業への追加ペナルティ
- `dataRegulation`: IT収益の上限緩和/抑制
- `rdSubsidy`: `rdStock` 成長率ブースト

### 7.2 Military政策
- `defenseProcurementCap`: 特定企業への集中契約抑制
- `exportControl`: Military収益抑制
- `warAudit`: 戦時の過大利益に課徴金

### 7.3 分配政策
- `dividendTaxRate` を配当分配に適用し、再分配原資へ。

## 8. 地政学相互作用
- 各国 `defenseBudget` は外交緊張・戦争ペア数で更新。
- Military企業の `lobbyPower` は緊張上昇に小さく寄与（上限あり）。
- 停戦・和平時は `defenseBudget` と Military需要を減衰させる。

## 9. UI/HUD表示拡張
- 企業サマリに追加:
  - 企業タイプ別社数（General/IT/Military）
  - 企業タイプ別利益シェア
  - HHI（集中度）と規制ペナルティ状態
- 地政学HUDに追加:
  - `defenseBudget` 平均
  - 戦時契約比率

## 10. 段階導入計画

### Phase 1（最小導入）
- `companyType` 追加
- IT/Military収益関数追加
- 採用選好追加
- 安全上限（profit/valuation cap）追加

### Phase 2（制度連動）
- antitrust, export control, procurement cap
- defenseBudget 更新ロジック連動

### Phase 3（再分配・安定化）
- dividend税・再分配
- 高集中時の自動政策発火（ガードレール）

## 11. 受け入れ基準（Acceptance Criteria）
- AC1: `companyType` 未設定既存データが壊れず実行可能。
- AC2: IT/Military導入後も 200日シミュレーションで NaN/発散なし。
- AC3: 企業利益上位1社のシェアが安全上限を超えない。
- AC4: 戦争発生時に Military収益上昇、停戦後に減衰する。
- AC5: 都市別失業率が許容範囲（既存比 +20%以内）に収まる。

## 12. 検証KPI
- 富集中: top1/top10 share, Gini
- 企業集中: HHI（都市/セクター）
- 地政学: war/crisis件数、平均緊張
- 雇用: 失業率、職業遷移
- 成長: 市場指数、企業利益総額

## 13. リスクと対策
- リスク: Military導入で戦争が常態化
  - 対策: warMultiplier上限、和平時減衰、監査ペナルティ
- リスク: IT勝者総取りで階層固定化
  - 対策: antitrust、配当税、R&D補助の都市偏在補正
- リスク: 指標の解釈混同（企業富 vs 個人富）
  - 対策: HUDに両方の比率を併記

## 14. 実装対象ファイル（想定）
- `src/sim/population.js`
  - `createCompany`, `applyCompanyLifecycle`, 収益計算部, 採用スコア部
- `src/sim/geopolitics.js`
  - `defenseBudget` 更新、緊張連動
- `src/config/defaultConfig.js`
  - `companyTypes`, `policy` 設定追加
- `web/main-app.js`
  - 企業タイプ表示・KPI表示

## 15. ロールバック条件
- 100日検証で以下のいずれか発生時は `companyTypes.enabled=false` で即時無効化:
  - 収益/評価額の発散
  - 戦争件数が既存比で異常増加（例: +200%以上）
  - 失業率の恒常的悪化（例: 3日移動平均で +10pt超）
